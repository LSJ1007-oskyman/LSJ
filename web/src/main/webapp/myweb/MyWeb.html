설명: 테이블사용법

사용하는 태그

  • <table>

  • <tr>

  • <th>

  • <td>

사용하는 속성

  • bgcolor

  • colspan

  • rowspan

추가적인 설명

  • 테이블 속성들

    • table : 테이블 생성

    • tr : 행(row)을 정의

    • td : 열 을 정의

    • border : 표의 테두리를 출력(값이 클수록 테두리의 두께가 두꺼워진다.)

    • width : 표의 가로길이(너비)

    • height : 표의 세로 길이(높이)

    • align : 표의 위치 - left, center, right

    • bgcolor : 표의 배경색 설정

    • cellspacing : 셀과 설 사이의 간격을 설정 ( 경계선의 두께)

    • cellpadding : 셀의 테두리와 셀 안의 내용간의 간격을 설정

  • colspan속성은 열 합치기기능으로 td, th 태그 안에서만 사용할 수 있다.

  • colspan 속성값은 몇개의 열을 합치는가에 따라서 숫자를 지정한다. 예) <td colspan = 3> : 3개의 열을 합치라는 의미

  • rowspan 속성은 행(row) 합치기 기능으로 td, th 태그 안에서만 사용할 수 있다.

  • rowspan 속성값은 몇개의 행을 합치는가에 따라서 숫자를 지정한다. 예)<td rowspan=3>: 3개의 행을 합치라는 의미

설명: 회원관리폼 만들기에 필요한 요소들

사용하는 태그

  • <form>

  • <input>

  • <caption>

  • <select>

  • <textarea >

추가적인 설명

  • form태그의 속성들

    • method : 값을 전달하는 방식을 설정하는 속성(get or post)

    • action : 값이 전달될 파일명을 설정

  • input태그의 속성들

    • size : 입력창의 크기 설정

    • maxlength : 최대 입력 가능한 글자수 설정

    • autofocus : 자동 포커스 기능

    • id : 각 양식의 태그를 구분하기 위한 역할

    • name : 값을 전단하기 위한 변수 역할 --넘거가는 곳(웹프로그램)에 id라는 변수에 값을 넘김

    • required : 입력 양식에 대한 필수 입력 검사

  • select 태그는 콤보 박스를 생성해주는 태그로서 option태그로 여러 요소를 나열해서 원하는것을 채크하는 방식이다.

  • select속성

    • autofocus: 페이지가 로드 되었을 때 해당 목록으로 바로 포커스가 맞춰진다.

    • disabled: 화면에는 보이지만 사용할 수 없게 만든다.

    • multiple: 여러 값을 선택할 수 있도록 한다.

    • name: 이름을 붙인다.

    • size: 목록을 펼치기 전에 보여질 목록의 개수를 정해준다.

    • selected: 이 속성이 추가된 요소를 미리 채크된 상태로 시작한다.

    • value: 서블릿으로 넘어갈 진짜 값을 입력하는 속성이다.

  • caption 태그는 테이블의 캡션(caption, 테이블이나 사진, 삽화 등에 붙는 설명)을 정의할 때 사용합니다.

  • table 요소는 단 하나의 caption 요소만을 명시할 수 있으며, caption 태그는 언제나 table태그 바로 다음에 위치해야 합니다.

  • caption 요소의 콘텐츠는 테이블 바로 위쪽에 가운데 정렬되어 표시되지만, CSS의 text-align이나 caption-side 속성을 사용하여 캡션의 위치나 정렬 방법 등을 변경할 수 있습니다.

설명: css diplay 속성

사용하는 속성

  • display :none,block,inline,inline-block

추가적인 설명

  • none으로 설정하면 안에 값이 보이지 않음.

  • block으로 설정하면 세로로 정렬된다.

  • inline으로 설정하면 가로로 정렬된다.

  • inline-block인 기본적으로 가로로 설정되며 안에 객체값 세부설정 가능하다.

설명: css background 속성

사용하는 속성

  • background-image: url()

  • background-size: 너비 높이

  • background-repeat: repeat-x,repeat-y,no-repeat

  • background-attachment: fixed,scroll

  • background-position: 키워드/background-position:x축/background-position:x축 y축

추가적인 설명

  • 기본적으로 body태그에 사용한다.

  • 기본적으로는 background는 repeat된다.

  • 각각에 그림을 layer라고 부른다.

  • size를 한개의 값으로 설정하면 너비로 인식하고, size를 두개의 값으로 설정하면 처음은 너비로, 두번째는 높이로 인식한다

  • size를 %로 설정하면 창에 크기에 맞게 사이즈가 맞춰진다.

  • repeat-x는 x축으로 반복,repeat-y는 y축으로 반복한다.

  • no-repeat속성을 사용하면 반복하지 않음.

  • background-attachment속성에 fixed를 사용하면 배경이 스크롤 내려도 고정되고, scroll로 하면 그 위치에 고정되어 스크롤 되어도 따라오지 않음.

  • background-position의 키워드 값으로 center,top,bottom이 있다.

설명: 미디어 쿼리

사용하는 속성

  • @media

  • @media sceen,print

  • @media (max-width: )(min-width: )

  • @media screen and (orientation: portrait,landscape)

추가적인 설명

  • 메타 태그에 뷰포트 설정을 통해서 단말기에 따라서 화면 스크린 출력 방식을 다르게 할 수 있다.(at_media_rule)

  • @media를 사용하여 단말기 종류에 따라서 출력 방식을 다르게 할 수 있다.

  • 위에 at_media_rule에서는 스크린상에 보여줄때와 프린트 할 때 각각 배경생을 다르게 하여 보여준다.

  • 외부 css파일로도 미디어속성에 종류에 따라서 다르게 출력할 수 있다.(media_attribute)

  • 픽셀값을 통해서 단말기의 종류를 파악해서 화면 출력값을 다르게 할 수 있다.(media_feature)

  • 오리엔테이션을 통해서 단말장치에 맞게 출력을 가로,세로 각각 따로 설정할 수 있다.portrait은 세로,landscape은 가로로 출력한다. (orientation)

  • 픽셀 크기에 따른 단말기에 맞게 메뉴와 컨탠츠의 배치를 설정한다.스마프폰 일때는 float값을 해제해서 아래로 정렬된다.(pattern1)

  • pattern2는 pattern1과는 다르게 오른쪽에 메뉴가 있다.

  • pattern3,4는 pattern1,2와는 다르게 body배열순서를 컨탠츠를 먼저하고 메뉴를 나중에 해서 스마트폰 일때는 float값을 해제되어 있어서 컨탠츠가 먼저 출력된다.

설명: javascript기초

추가적인 설명

  • window객체는 브라우저 관련 최상위 내장객체로서 보통은 생략을 한다.

  • 문자를 출력할때 쌍따옴표와 외따옴표를 모두 사용할 수 있다.

  • 문장의 끝에는 세미콜론을 생략할 수 있다.

  • 출력문 안에 태그값을 넣을 수 있다.

  • writeln은 줄바꿈을 해주지 못한다.

  • 자바스크립트의 주석은 java와 동일하다.

  • 이클립스 source에 toggle comment로 주석처리가 된다.(ctrl+shift+c)

  • 여러가지 출력 방법이 존재한다.(first)

  • 외부 js파일을 불러올 수 있다.(call)

  • 선언자 없이 변수 선언하면 전역변수가 된다.

  • 변수명은 대소문자를 구분한다.

  • 변수명은 영문자,숫자,$,_로 만들 수 있다.

  • 변수명의 첫 글자는 숫자로 시작할 수 없다.

  • 예약어로 설정된 단어들은 변수명으로 사용할 수 없다.

  • var선언자는 변수선언, 변수선언 후 동시에 값할당, 한번에 여러개의 값 할당, 재할당이 가능하다.(var)

  • let선언자는 변수선언, 변수선언 후 동시에 값할당, 한번에 여러개의 값 할당은 가능하나 재할당은 불가능하다.(let)

  • const선언자는 변수선언, 변수선언 후 동시에 값할당, 한번에 여러개의 값 할당은 가능하나 변수 선언과 동시에 반드시 초기값, 즉 초기화를 해야하며 재할당은 불가능하다.(const)

  • const선언자는 주로 상수를 만들때 사용하는 선언자이다.

  • 변수 출력할시 변수를 넘겨줄때 ,또는 + 둘다 사용가능하다.(variable)

  • javascript에 자료형이 없는 것은 아니다.단지 변수선언때 사용하지 않을 뿐이다.(variable)
